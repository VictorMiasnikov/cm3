Error Handling Statements

You can use exceptions to communicate unexpected problems to a client. Exceptions are a much safer way of dealing with errors and exceptional conditions than return codes.

  • RAISE Statement: Raises an exception, passing control to the exception handler.
  • TRY-EXCEPT Statement: Executes the TRY clause. In case of an exception, will try the exception handlers.
  • TRY-FINALLY Statement: Executes the TRY clause. Executes the clean-up code no matter what happens in the main body.
